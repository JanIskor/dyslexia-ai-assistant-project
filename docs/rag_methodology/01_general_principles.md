# Общие принципы адаптации текста для обучающихся с дислексией

## Назначение документа

Этот документ задаёт базовые правила для любой адаптации. Его chunks подходят как общий методический контекст для RAG, когда системе нужно выбрать безопасный способ упрощения текста без потери учебного смысла.

## Ключевая цель адаптации

### Сохранять смысл

- Адаптация не должна искажать содержание исходного текста.
- Факты, имена, даты, формулы, названия и термины нужно сохранять.
- Если информация учебно важна, её нельзя удалять только ради краткости.

### Снижать когнитивную нагрузку

- Убирать лишнюю сложность формы, а не упрощать предметное содержание до потери смысла.
- Делать текст более предсказуемым для чтения.
- Помогать читателю быстрее находить главное.

## Уровни адаптации

### Лексический уровень

- заменять редкие и перегруженные слова на более частотные, если смысл сохраняется;
- пояснять термин, если без него нельзя обойтись;
- не подменять предметный термин бытовым словом, если это разрушает учебную точность.

### Синтаксический уровень

- сокращать слишком длинные предложения;
- убирать лишнюю вложенность;
- делать причинно-следственные связи явными.

### Визуальный уровень

- разбивать текст на короткие блоки;
- выделять смысловые опоры;
- не перегружать фрагмент избыточным форматированием.

## Ограничения безопасности

### Что обязательно сохранять

- фактическую точность;
- авторскую логическую последовательность, если она учебно значима;
- предметно критичные термины;
- формулировки, где точность важнее удобства чтения.

### Что нельзя делать

- добавлять новые факты из внешних знаний;
- компенсировать непонятность домыслами;
- удалять важные условия, причины или исключения;
- смешивать адаптацию формы с изменением содержания.

## Жанровое ограничение

- Перед переписыванием учитывать жанр исходного текста.
- Чем выше риск искажения формулировки, тем осторожнее должна быть адаптация.
- Для чувствительных жанров сначала выбирать более щадящий формат подачи, а не глубокое переписывание.

## Когда применять

- когда нужен общий набор безопасных правил для любой адаптации;
- когда retrieval должен вернуть базовые guardrails по сохранению смысла;
- когда система работает с новым текстом без узкоспециализированных жанровых указаний.

## Когда не применять

- когда нужен только узкий набор правил по одному уровню адаптации, например только лексика или только визуальная разметка;
- когда задача относится к предметному содержанию, а не к форме подачи;
- когда требуется юридическая, нормативная или иная дословная точность без переписывания.

## Связанные режимы адаптации

- `basic_simplify`
- `structured_explanation`
- `key_points_focus`
